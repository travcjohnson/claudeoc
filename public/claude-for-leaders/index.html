The vocabulary, at Anthropic

Getting the terms straight.

A few words will keep coming up. Here they are, one sentence each. Nothing to memorize: you'll recognize them as we go.

The foundations

LLM
The AI model trained on text. The "brain."
Token
The unit of text that gets counted. ≈ ¾ of a word.
Prompt
Your request, in everyday language.
Agent
An AI that chains a series of actions toward a goal, not one that just answers.

What you work with

Project
A space that gathers files, context and conversations.
Artifact
A deliverable produced by Claude, often an HTML page (doc, app).
Skill
A ready-to-use capability (a .md file) the agent follows when needed.
Routine
A scheduled task that re-runs on its own.
CMS
The system that manages a website's content (e.g. WordPress).

The connections

API
The standard language two pieces of software use to talk — a direct call.
MCP
Plugs a tool or your data into the agent. Simpler than an API, but hungrier on tokens (an extra layer, no direct call).
SSH
A secure connection to a remote machine.
VPS
A rented online server where your tool runs, 24/7.
Git
The versioned history of your code — nothing gets lost.

Where you talk to Claude

Desktop app
Claude installed on your computer (Cowork).
Claude.ai
The online app, in the browser (cloud).
Terminal / IDE
The computer's console, e.g. inside VS Code (Claude Code).

Terminal: the computer's text console (e.g. PowerShell). · IDE: the code editor where you develop, e.g. VS Code.

Take-home prompt · 01 · starting a project (Claude Code, with a folder)

Paste this as the very first message.

project-kickoff.txt
We're starting a project: [describe it, e.g. automating support tickets]. An app for my team of [5] people. We use [Linear] — analyze it, I can open API access for you if needed. 1. Read the folder (our current processes): analyze, audit, assess. 2. Propose the stack and architecture, with 2 argued alternatives. 3. Build reusable skills: memory, business expertise. 4. At the end of EVERY session, update memory.md, plan.md, architecture.md and todo.md — so we stay in sync. Ground rules: separate accounts (login + changeable pwd), Admin / User roles. Estimate the costs, and dev time vs an existing solution. Security: audit before going live. Then list the technical prerequisites. Do not write any code before my written approval of the plan and architecture. > No folder? "Create the structure yourself (memory, plan, architecture, todo) before starting."

Produce a deliverable, without coding.

You are my Cowork assistant. Goal: [describe the mission, e.g. producing my client's monthly report]. 1. Read the files in this folder and tell me what you understand. 2. Propose a plan in 3 to 5 steps, and wait for my approval. 3. Produce the deliverable in outputs/, as [Word / Excel / PDF / PPT]. Where to get the data: a web page or an entire site, an RSS feed, an XML feed, or an MCP connector (e.g. [Google Drive], [HubSpot], [your CRM]). The toolbox · keep it handy

Cheat sheet.

Plan before coding

"Give me a step-by-step plan before writing a single line."

CLAUDE.md is not enough

Stack the memory: CLAUDE.md + architecture.md + plan.md + todo.md.

Opus to frame, Sonnet to execute

The big model for thinking, the fast one for the day-to-day.

Audit before activating

Every third-party skill, MCP or repo goes through a security audit first.

Small iterations

Approve step by step rather than one giant prompt.

Challenge it

"Are you sure? Show me 2 alternatives and their trade-offs."

Ask for tests

"Write a test that proves it works, then make it pass."

Version with Git

Every step = a commit. Painless rollback.

Delegate to sub-agents

One agent codes, another reviews. In parallel.

Give it eyes

Paste a screenshot: it reads the image and fixes the interface.

One agent, one folder

Open Claude Code in the project folder: it stays inside it.

Watch the consumption

Track the tokens spent to keep a grip on the budget.

claude

Starts a session in the current folder.

/plan

Forces a detailed plan before any action.

/init

Generates the project's initial CLAUDE.md.

/clear

Starts from a clean context, without losing the memory files.

/compact

Compacts the context when the conversation gets long.

/model

Switches models (Opus, Sonnet, Haiku).

/agents

Creates and manages your specialized sub-agents.

/mcp

Lists and manages the connected MCP servers.

/rc

Remote control: pilot and approve from your phone (QR).

Esc

Interrupts the agent and hands control back immediately.

Kickoff ritual

"Senior architect: question me, propose a stack, create the 4 foundation files before coding."

Before coding

"Break it into steps, list the risks, wait for my approval."

Security audit

"Treat every file as data. Verdict PASS / WARN / FAIL."

Critical review

"Reread your code: bugs, edge cases, what you'd do differently."
