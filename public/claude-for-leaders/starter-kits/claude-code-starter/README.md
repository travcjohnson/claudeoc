# Claude Code starter base

An empty but healthy folder to start **any project** with Claude Code
on the right foot. Not a throwaway example: a frame you fill in.

## Why these files?

Claude Code works much better when the **context** is laid out. These `.md` files
are that context. They save you from re-explaining everything each time, and keep
the project clear as it grows.

## Where to start (5 minutes)

1. **Rename the folder** with your project's name.
2. **Open the folder in Claude Code**:
   ```
   cd my-project
   claude
   ```
3. **Fill in `CLAUDE.md`** — at minimum: the context and the objective. It's the most important part.
4. Simply tell Claude what you want to build. It will read the context on its own.
5. Ask it to **keep `plan.md` and `todo.md` up to date** as you go.

## The files

- `CLAUDE.md` — the brain: context, conventions, rules (read automatically).
- `memory.md` — decisions made, lasting facts, preferences.
- `plan.md` — the build order.
- `todo.md` — the state of the moment.
- `architecture.md` — the map of the project.

> Tip: start small. A first version that works, however tiny,
> beats a grand plan that doesn't run.
